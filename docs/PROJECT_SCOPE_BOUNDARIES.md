# حدود نطاق المشروع (Project Scope Boundaries)

## القائمة (أ): ما يمكننا إنتاجه بكفاءة 100%
- **مواقع الهوية الشخصية (Portfolios):** للمهندسين، المصورين، والكتّاب.
- **مواقع الخدمات المحلية:** مطاعم، مقاهي، عيادات، مكاتب محاماة.
- **صفحات الهبوط التسويقية (Landing Pages):** لترويج منتج واحد أو خدمة بسيطة.
- **مواقع المعارض البسيطة:** معارض فنية أو كتالوجات منتجات ثابتة (Static).

## القائمة (ب): ما هو خارج النطاق (الممنوعات)
- **منصات التجارة الإلكترونية الضخمة:** (مثل أمازون أو نون) التي تتطلب أنظمة سلة تسوق وإدارة مخزون معقدة.
- **بوابات الدفع البنكي المباشر المخصصة:** (نكتفي بتكامل بسيط مع PayPal/Stripe للمواقع المدفوعة مسبقاً).
- **مواقع الأخبار الديناميكية المتغيرة لحظياً:** التي تحتاج لأنظمة CMS معقدة جداً.
- **تطبيقات التواصل الاجتماعي:** (أنظمة دردشة، ملفات شخصية للمستخدمين، منشورات).

### التبرير المنطقي (The Logic):
الهدف من GetYouSite هو **"السرعة والجودة البصرية"**. الأنظمة في القائمة (ب) تتطلب إدارة قواعد بيانات خلفية (Backend) صعبة الصيانة والتأمين، والمخاطرة فيها قد تكسر تجربة المستخدم البسيطة التي نعد بها. نحن نبني "أصولاً رقمية" (Digital Assets) وليست "منظومات برمجية" (Software Ecosystems).
