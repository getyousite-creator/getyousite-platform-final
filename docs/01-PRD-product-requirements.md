> **الحالة:** وثيقة مرجعية إلزامية
> **الإصدار:** 2.0

---

## 1. الرؤية الجوهرية

**GetYouSite** هي منصة ثورية تحوّل الوصف النصي الطبيعي — حتى لو كان عاميًا أو غير تقني — إلى موقع ويب كامل، وظيفي، وجميل، دون أن يكتب العميل سطرًا برمجيًا واحدًا.

---

## 2. (أ) ما الذي يفهمه الذكاء الاصطناعي من وصف العميل

### المدخلات المستخلصة تلقائيًا:

| الفئة | ما يتم استخلاصه | مثال |
|--------|-----------------|------|
| **النشاط التجاري** | نوع العمل، القطاع، الحجم | "مطعم عائلي للمأكولات البحرية" → قطاع: مطاعم، حجم: صغير |
| **الجمهور المستهدف** | الفئة العمرية، الاهتمامات، الموقع الجغرافي | "عائلي" → عائلات، أطفال، أجواء دافئة |
| **النغمة (Tone)** | رسمي / ودود / عصري / فاخر / مرح | "عائلي" → ودود ودافئ |
| **الوظائف المطلوبة** | نماذج، حجز، معرض، مدونة، متجر بسيط | "حجز طاولات" → نموذج حجز تفاعلي |
| **الهوية البصرية** | ألوان، أنماط، إلهام بصري | "ألوان البحر" → تدرجات زرقاء وفيروزية |
| **المحتوى** | نصوص، صور مطلوبة، عناصر خاصة | "صورة للطهاة" → قسم فريق العمل |

### آلية الاستخلاص:
```
وصف العميل → تحليل NLP (GPT-4o) → JSON منظم → محرك التصميم
```

```json
}
```

## 3. (ب) المخرجات الملموسة

### ما ينتجه النظام فعليًا:

| المخرج | الوصف | التنسيق |
|--------|-------|---------|
| **هيكل الصفحات** | خريطة موقع كاملة مع الروابط الداخلية | JSON Blueprint |
| **التصميم البصري** | تخطيط فريد يتناسب مع طبيعة النشاط | React Components + CSS |
| **المحتوى المكتوب** | نصوص ترحيبية، عناوين، أوصاف خدمات — بالعربية الفصيحة | Localized Strings |
| **الصور** | صور مناسبة من مكتبة مرخصة أو مولدة بالذكاء الاصطناعي | WebP/PNG |
| **الأكواد** | ملفات HTML/CSS/JS جاهزة للنشر | Static Export أو Vercel Deploy |
| **ملف SEO** | عناوين meta، وصف، بيانات منظمة JSON-LD | Head Tags |

### البنية المعمارية للمخرج:
```
site-output/
├── index.html          ← الصفحة الرئيسية
├── pages/
│   ├── about.html      ← من نحن
│   ├── menu.html       ← قائمة الطعام
│   ├── reservation.html ← الحجز
│   └── contact.html    ← اتصل بنا
├── assets/
│   ├── css/style.css   ← تصميم متجاوب
│   ├── js/main.js      ← تفاعلات
│   └── images/         ← صور محسنة
├── manifest.json       ← PWA
├── sitemap.xml         ← خريطة الموقع
└── robots.txt          ← توجيهات محركات البحث
```

## 4. (ج) الحدود الواقعية — ما لا نعد به

### ✅ ما نعد به بثقة:
- مواقع تعريفية جميلة للمشاريع الصغيرة والمتوسطة
- محتوى ذكي مكتوب بلغة سليمة
- تصميم متجاوب يعمل على جميع الأجهزة
- نشر فوري على نطاق فرعي أو خاص
- محرر بصري ذكي للتعديلات

### ❌ ما لا نعد به (بصراحة مطلقة):
- **لا نعد** بمتجر إلكتروني معقد كأمازون (نوفر صفحة منتجات بسيطة فقط)
- **لا نعد** بموقع أخبار ديناميكي مع CMS كامل
- **لا نعد** بتصميم ثلاثي الأبعاد معقد من وصف قصير
- **لا نعد** ببوابة دفع مالي مباشر (نقدم تكامل مع Stripe/PayPal)
- **لا نعد** بتطبيق ويب تفاعلي (SaaS) كامل

### رسالة الشفافية للعميل:
> "GetYouSite مثالية للمشاريع الصغيرة والمتوسطة التي تحتاج حضورًا رقميًا احترافيًا وسريعًا. إذا كنت تبحث عن منصة تجارة إلكترونية معقدة أو تطبيق ويب متقدم، ننصحك بالتعاون مع مطور متخصص."

## 5. مؤشرات النجاح (KPIs)

| المؤشر | الهدف |
|--------|-------|
| زمن إنشاء موقع من الوصف | < 90 ثانية |
| رضا العميل عن المخرج الأول | > 70% "مقبول بدون تعديل" |
| جودة المحتوى المُولد | > 85% لا يحتاج تعديل يدوي |
| معدل التحويل (تجربة → اشتراك) | > 15% |
| وقت تعلم المحرر | < 5 دقائق |

## 6. المتطلبات التقنية الأساسية

### Stack التقني:
- **Frontend:** Next.js 16 + React 19 + TypeScript + Tailwind CSS
- **AI Engine:** OpenAI GPT-4o (تحليل + توليد محتوى)
- **Database:** Supabase (PostgreSQL) + Prisma ORM
- **Auth:** Supabase Auth (OAuth + Email)
- **Payments:** Stripe + PayPal
- **Deployment:** Vercel (Edge Functions + ISR)
- **i18n:** next-intl (العربية كلغة أولى)

### متطلبات الأداء:
- LCP (أكبر عنصر محتوى): < 2.5 ثانية
- FID (تأخر التفاعل الأول): < 100 مللي ثانية
- CLS (إزاحة التخطيط): < 0.1
- حجم الحزمة الأولى: < 200 كيلوبايت
