# إستراتيجية التوليد التكيفي والمحتوى الذكي

## 1. محرك التصميم التكيفي (Adaptive Layout Engine)
سيقوم النظام بتحويل "نوع النشاط" إلى رموز تصميم (Design Tokens):

| نوع النشاط | كثافة الصور | حجم الخطوط | التباعد (Spacing) | الحيوية (Vibrancy) |
| :--- | :--- | :--- | :--- | :--- |
| م مكتب محاماة | منخفضة | متوسط/رسمي | ضيق/منظم | منخفضة (Classical) |
| معرض فني | عالية جداً | كبير/فني | واسع جداً | عالية (Vibrant) |
| مطعم | متوسطة | جذاب/عريض | مائل للتوسيع | دافئة (Earth/Warm) |

## 2. بروتوكول توليد المحتوى (Text AI Prompting)
سيُمنع استخدام أي نص وهمي. إليك الهيكل البرمجي لتوليد المحتوى:

### مدخلات المحرك (Inputs):
- `Project_Name`: الاسم
- `Industry`: النشاط
- `Tone`: النغمة (رسمية، شبابية، إلخ)

### المخرجات المطلوبة (Outputs):
1. **Hero Text:** عنوان يخاطب المشكلة أو الرغبة الأساسية للعميل.
2. **Value Prop:** فقرة ترحيبية (100 كلمة) تشرح "لماذا نحن؟".
3. **Services Titles:** اقتراح خدمات منطقية بناءً على النشاط (مثلاً لمطعم: "حفلات خاصة"، "توصيل منزلي").
4. **Arabic Localization:** ضمان استخدام فصحى سليمة أو عامية بيضاء مهذبة حسب الطلب.

### مثال لبرومبت داخلي للذكاء الاصطناعي:
> "أنت كاتب محتوى إبداعي. الاسم: 'مكتب الصفوة للمحاماة'. النشاط: 'قضايا عمالية'. النغمة: 'رسمية وموثوقة'. ولّد عنواناً رئيسياً ونبذة عن الخدمات باللغة العربية الفصحى."
